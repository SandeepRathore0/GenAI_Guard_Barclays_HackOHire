# GenAI Guard Setup & Integration Plan (v3)

## Goal Description
The user wants to improve the accuracy of all the guard models to avoid false positives. They want to train new robust models and implement a fallback mechanism using the HuggingFace Inference API to ensure high-accuracy classification across all guards.

## Proposed Changes

### 1. HuggingFace API Integration as Fallback
To prevent false positives and provide state-of-the-art verification, we will integrate HuggingFace's Inference API.
#### [MODIFY] `app/core/model_loader.py`(file:///d:/Project/GenAi-Guard/GenAi-Guard/app/core/model_loader.py)
- Create a new method `huggingface_predict(payload, api_url)` that sends inference requests to a configured HuggingFace endpoint.
- Modify the `predict` wrapper so that if the local model prediction returns a positive (threat) or if the local model is uncertain, it dispatches a secondary verification check to HuggingFace.

#### [MODIFY] `app/core/config.py`(file:///d:/Project/GenAi-Guard/GenAi-Guard/app/core/config.py)
- Add `HUGGINGFACE_API_KEY` to the settings (to be read from an `.env` file).

### 2. High-Accuracy Model Training pipeline
We will create a robust script to train machine learning models using better datasets (e.g., standard NLP datasets available through `datasets` library, specifically Enron/Spam datasets).
#### [NEW] `train_advanced_models.py`(file:///d:/Project/GenAi-Guard/GenAi-Guard/train_advanced_models.py)
- **Text Guard**: Fetch a recognized spam/phishing dataset from the Huggingface `datasets` hub (e.g., `SetFit/enron_spam` or similar), train a robust Random Forest or XGBoost model with TF-IDF, and store the vectorizer and model.
- **Web Guard**: Generate a more comprehensive URL dataset with known phishing patterns and safe banks, and train an ensemble model (Random Forest) instead of a simple tree or naive bayes model to reduce false positives.

### 3. Update Guard Routes for Fallbacks
#### [MODIFY] `app/modules/text_guard/routes.py`(file:///d:/Project/GenAi-Guard/GenAi-Guard/app/modules/text_guard/routes.py)
- Update the phishing detection logic: Use `ModelLoader.huggingface_predict()` (e.g., calling `protectai/deberta-v3-base-prompt-injection` or similar) to verify high-risk text.

## User Review Required
> [!IMPORTANT]
> To use the HuggingFace Fallback, you will need a **HuggingFace API Key (Access Token)**. Please generate one at `https://huggingface.co/settings/tokens` and let me know the key or manually add it to an `.env` file in the project directory (`HUGGINGFACE_API_KEY="hf_..."`).

> [!NOTE]
> Training robust models for *Audio Deepfake* and *File Verification* locally is highly dependent on massive datasets (GBs of data). For this hackathon/project scope, I propose we stick to lightweight models for Web/Text and rely strictly on HuggingFace zero-shot classification pipelines for the complex File/Audio analysis if possible. We will focus initially on Text and Web model improvements.
